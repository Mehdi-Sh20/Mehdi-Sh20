

# 📚 Book List Management (React + Vite)

A modern React application for managing a list of books.
Users can add, view, edit, and delete books through an interactive and responsive interface powered by a local API.

This project demonstrates real-world React concepts, including component architecture, asynchronous data handling, UI state management, and clean design patterns.

---

## 🚀 Features

* Fetch and display books from a local API
* Add new books via a modal form
* Edit existing book details
* Delete books from the list
* Toggle book list visibility (Show / Hide)
* Loading indicator during data fetching
* Error handling for API requests
* Dynamic status messages (add / edit / delete)
* Fully responsive design (mobile-friendly)
* Automatic UI updates using React state
* Keyboard navigation with arrow keys and active item highlighting

---

## 🛠 Technologies Used

* **React (v19.2.4)**
* **Vite**
* **JavaScript (ES6+)**
* **CSS Modules**
* **Fetch API**
* **JSON Server (Local API)**

## ⚙️ Installation & Setup

### 📌 Environment Versions

| Tool          | Version  |
| ------------- | -------- |
| Node.js       | v22.12.0 |
| React         | v19.2.4  |
| npm           | v10.9.0  |

---

### 1️⃣ Clone the Repository

```bash
git clone https://github.com/Mehdi-Sh20/books-list-management.git
cd books-list-management
npm install
```

---

### 2️⃣ Run JSON Server (Local API)

```bash
npx json-server --watch data/db.json --port 3000
```

---

### 3️⃣ Start the Development Server

```bash
npm run dev
```

Then open in your browser:

```
http://localhost:5173/
```

## 🧠 Key React Concepts Demonstrated

### 🔹 State Management

Using `useState` to manage UI and application data.

### 🔹 Side Effects

Using `useEffect` for data fetching and synchronization.

### 🔹 Async/Await

Handling asynchronous API calls in a clean and readable way.

### 🔹 Error Handling

Using `try/catch` for safe API operations.

### 🔹 Conditional Rendering

Rendering UI based on:

* Loading state
* Error state
* Empty data
* Success messages

### 🔹 Component Communication

Using:

* `forwardRef`
* `useImperativeHandle`

To trigger actions (like notifications) from parent components.

### 🔹 Custom Hooks

Using a custom hook (`useKeyboardNavigation`) for reusable keyboard navigation logic.

---

## 🔄 Application Flow

1. Books are fetched from the API
2. A loading spinner is displayed during data fetching
3. Users can:

   * Add a new book (via modal)
   * Edit an existing book
   * Delete a book

4. After each action:

   * A notification message is shown
   * The UI updates automatically
